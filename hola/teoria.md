## IDE
1. ¿Qué es un IDE y para qué sirve?

Integrated Develpment environment, es un sfotware que desarrolla otro software, en el cual se puede, compliar, debugger, ejecutar los codigos, pueden tener extensiones, IA, editores de texto, algunos son multilenguaje.
2. ¿Qué herramientas suelen incluir los IDEs modernos?

Extensiones, debbugers, compiladores, IA, autompletado de codigo, diseño de interfaz, identificacion del error en cierto punto.

3. ¿Cómo surgieron los primeros IDEs y cuál fue el primero?

En 1964 por Jhon kemeny y Thomas Kurtz, creacon el BASIC DARTHMOUTH 
Sin embargo otros mencionan que el primer ofue Maestro I, ya que incluía debuggers. 

4. ¿Qué aportó Dartmouth BASIC en 1964 al desarrollo de los IDEs?
La creaccion e innovacion de nuevos IDEs un futuro después, siendo que los IDE, tuvieran compiladores, debuggers, etc.

5. ¿Qué diferencia existe entre un IDE por lenguaje y un IDE multilenguaje?

Que el IDE lenguaje está diseñado para un solo lenguaje, por ejemplo INTELLj dea con java o apache también
y un multilenguaje sería algo como visual studio code


6. Ventajas y desventajas de IDE y editores de texto

| IDE | Ventajas | Desventajas |
| --- | :--------------- | :------- |
| --- | Tienen compliladores, debuggers, buena gestión y organizaión de proyectos de manera profesional, control de versiones integrado, detección de errores en tiempo real, autocompletado inteligente, IA | Consumo alto de memoria, arranque lento, pueden ser de pago, interfaces complejas, curva de aprendizaje pronunciada | 
| EDITOR DE TEXTOS | Arranque rápido, interfaz sencilla, consumo minimo de memoria, personalización total, funciona en hardware limitado | Sin autocompletado inteligente, no son realmente utiles para una gestion de proyectos profesional, depuración de código manual, no tienen extensiones, no tienen debuggers. |
